# VehTrust 严重度模型与深度学习流程说明

> 面向项目内“事故严重度分析”链路的完整说明：数据来源、训练流程、端侧推理、代码入口与可扩展路线。

---

## 一句话结论

- 当前严重度页面已切换为**本地推理链路**，不依赖远端严重度 API。  
- 现阶段训练主模型为 **LogisticRegression（结构化表格模型）**，不是神经网络。  
- Python 训练产物会导出为 Android 可直接加载的 JSON 参数，在端侧完成概率计算与解释展示。

## 严重度标签定义

训练脚本 `backend/ml/train_collision_severity.py` 中采用三分类：

- `1 -> Fatal`
- `2 -> Serious`
- `3 -> Slight`

---

## 训练主流程（ML）

### 1) 读取与清洗

- 读取 collision 主表，构造时间衍生特征（月份、小时、周末、夜间等）
- 读取 vehicle 明细表，处理驾驶员年龄、排量、车龄等数值字段

### 2) 事故级聚合

vehicle 表按 `collision_index` 聚合为事故级特征（例如车辆数量、驾驶员年龄统计、车辆类型众数等）。

### 3) 特征拼接与防泄漏

- collision 与 vehicle 聚合表按 `collision_index` 合并
- 去除目标泄漏字段与不应直接入模的字段
- 数值列与类别列分开进入预处理管线

### 4) 预处理与训练

- 数值：`SimpleImputer(median)` + `StandardScaler()`
- 类别：`SimpleImputer(most_frequent)` + `OneHotEncoder(...)`
- 模型：`LogisticRegression(solver="saga", class_weight="balanced", max_iter=800)`

### 5) 评估与落盘

训练后输出：

- `backend/ml/artifacts/collision_severity_pipeline.joblib`
- `backend/ml/artifacts/collision_severity_metrics.json`

---

## 为什么 Android 不能直接用 joblib？

`joblib` 是 Python/sklearn 序列化格式，Android 端无法直接加载执行。  
因此项目通过导出器做“端侧运行时参数化”：

- 训练后执行 `backend/ml/export_collision_severity_runtime.py`
- 生成 `app/src/main/assets/collision_severity_model.json`

这个 JSON 包含 `base_logits`、数值缩放参数、类别增量参数等，足够端侧独立完成三分类打分。

## Android 端推理链路（本地）

关键入口：

- `app/src/main/java/com/example/vehtrust/trace/CollisionSeverityApi.kt`
- `app/src/main/java/com/example/vehtrust/trace/AccidentTraceViewModel.kt`
- `app/src/main/java/com/example/vehtrust/AccidentTraceDetailActivity.kt`

执行逻辑：

1. ViewModel 收到事故详情包
2. `CollisionSeverityApi` 从 `assets/collision_severity_model.json` 读取运行时参数
3. 计算 logits → softmax 概率（Fatal / Serious / Slight）
4. 页面展示：摘要、概率、关键特征、解释文本

## 可演进路线（深度学习版本）

如果后续要将严重度模块升级为深度学习，可按以下路线推进：

1. 保留当前 LogisticRegression 作为稳定 baseline
2. 新增 `train_collision_severity_torch.py`（或等价脚本）
3. 输入扩展为“事故前 10 秒时序遥测 + 环境离散特征 + 车辆聚合特征”
4. 选择 LSTM / GRU / 1D-CNN / Transformer Encoder
5. 导出为 TFLite（优先）或 ONNX，并在 `CollisionSeverityApi` 中切换推理后端

## 当前方案边界

- 已经是“训练模型参数驱动”的本地推理，不是纯手工规则。  
- 但端侧执行的是导出后的压缩参数子集，不是 sklearn 全管线原样搬运。  
- 若要求 100% 复刻全部训练特征，需要进一步扩展车端采集字段与导出器能力。
